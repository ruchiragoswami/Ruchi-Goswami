Difference between <strong> and <b> tags in HTML

August 2021

In our daily lives, we often come across two words: Strong and Bold, they both have different meanings. Strong is something that can produce or withstand great physical force while bold is daring or courageous. In HTML also these words exist as tags and yes, they have different meanings.

While writing content in HTML, many times we have to use bold fonts. Reasons for using bold texts can be different. Most of the times, we just want to draw attention of the users. Sometimes, the text itself has more importance than the surrounding text and therefore, we need to make it bold.

Using <strong> tag

The bolded text has more importance than the surrounding text.
For example, "This text is < important > ! And this text is not important. "
This tag is used when certain text needs to be emphasized to make it stand out.

Using <b> tag

Use <b> tag when you just want to draw the user's attention, or to make the text appear logically different or aesthetically pleasing. This tag is used when text needs to be bolded for styling purpose only.

To conclude, for styling/presentaion <b> is your 'go-to' tag, for context (to the browser), <strong> is your tag. Although both the above tags will give same results, under the hood they are different. It is not advisable to use them interchangeably because of their sematics. Using semantic tags is important for screen readers as these tags add meaning to the markup. As a result, search engines, screen readers and web browsers can make sense of it. To understand this, let us look at the example below:
If a user is using a text-to-speech program, then all the text is spoken to them. If <strong> tag is used, the text-to-speech program will speak with some emphasis on that word giving the user an idea that the word was important. But if a <b> tag is used, the text-to-speech program will not speak it differenly, and the user would never get a clue that the word was important. In HTML, semantics is an important thing.
